## 程序功能    用于批量发送数据到kafka，可以控制发送的频率和内容。常用于压测和集成测试。 ## 部署方法    设置环境变量中的JAVA_HOME，注意JAVA版本至少为1.8。      也可以不设置环境变量，但需要手工指定run.bat中的JAVA_HOME变量。## 基本使用方法    1. 修改example_start.bat中的参数，调整发送的频率和发送的kafka地址。            thread_num=3  //使用几个线程发送给kafka            number_per_batch=1000 //每个batch发送几条数据。  注意batch之间的间隔固定为1秒。            bootstrap_servers=localhost:9092 rem "182.180.115.73:9092,182.180.115.74:9092,182.180.115.75:9092";  //要发送的kafka服务器地址            kafka_topic=sparktest //要发送的kafka topic            config_file=data.cdd //设置要执行的配置文件名称                        按照上述的配置，每秒钟发送的数据量是3x1000=3000个。                2. 修改config_file对应的文件，设置模拟的数据源。    3. 再执行example_start.bat运行程序    4. 可以打开log.log查看日志。            注：如果想要一次执行多个脚本，可以拷贝一份example_start.bat，将里面的config_file设置为你想要的文件。        config_file文件样例    10 | {"name":"11", "value":100}               ―― 重复发送{"name":"11", "value":100}到kafka，一共发送10次 "value1":[[1, +10, 20]]}         ―― 以{"name":"11", "value":0, "value1":1}                                                                                   {"name":"11", "value":-10, "value1":11} +100]]}         ―― 类似上面的逻辑，但发送的value是在10和100之间的随机数。其中第一次发送的数据value值肯定是10.                                                                                     1 |
